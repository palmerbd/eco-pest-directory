ballroomdancedirectory.com/competitions · Mirrors the studio listing → claim → dashboard → featured pattern exactly
🎯 What We're Building

A full Competitions section built on the same four-layer architecture as the studios — public listings → claim → dashboard → featured upgrade. Competition organizers follow the same funnel studio owners use: their event is already listed, they claim it for free, and they can upgrade to Featured to advertise on city pages and drive registrations.

By mirroring the studio pattern precisely, we reuse existing Supabase auth, Stripe checkout, Resend email, ISR revalidation, and dashboard patterns. The net result is ~50% less code compared to building this from scratch, with no rework on shared infrastructure.

Key revenue insight: Competition organizers have direct financial incentive to pay for city page advertising — a $199/year Featured listing that puts their competition in front of thousands of dancers in their city is an easy sell. One Featured competition organizer covers months of server costs.

🗄️ New Supabase Tables (Migrations 005 + 006)
Two new tables. Both mirror existing patterns exactly — no new architectural patterns required.
Migration 005 — competition_claims
-- Mirrors: claims table
CREATE TABLE competition_claims (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  competition_slug text NOT NULL,
  competition_name text NOT NULL,
  organizer_name text NOT NULL,
  organizer_email text NOT NULL,
  organizer_phone text,
  user_id uuid REFERENCES auth.users(id),
  status text DEFAULT 'pending' CHECK (status IN ('pending','verified','approved','rejected')),
  tier text DEFAULT 'claimed' CHECK (tier IN ('claimed','paid')),
  stripe_customer_id text,
  stripe_subscription_id text,
  created_at timestamptz DEFAULT now()
);
-- RLS: service-role full access, owner can read own row
Migration 006 — competition_profiles
-- Mirrors: studio_profiles table (Migration 004)
CREATE TABLE competition_profiles (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  competition_slug text UNIQUE NOT NULL,
  claim_id uuid REFERENCES competition_claims(id),
  logo_url text,
  custom_description text,
  facebook_url text,
  instagram_url text,
  tiktok_url text,
  youtube_url text,
  promo_text text,
  promo_type text CHECK (promo_type IN ('percentage','dollar','free_trial','custom')),
  promo_savings text,
  promo_end_date date,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
-- RLS: service-role full access, anon public read
