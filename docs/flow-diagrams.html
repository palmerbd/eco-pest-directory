Part B — The Backend Handoff (what happens in the background)
5
Stripe fires a webhook to our server
Stripe
Vercel
The moment Stripe confirms the payment, it sends an HTTP POST to our site at /api/stripe/webhook. This is the authoritative confirmation — this is where we trust that money changed hands and we take action. The webhook includes a secret signature we verify to ensure it's really from Stripe, not a fake.
🆕New API route to build: /api/stripe/webhook
🔐First thing it does: verify the Stripe webhook signature — rejects anything unsigned
📦Event types we handle: checkout.session.completed (new sub), customer.subscription.deleted (cancellation), invoice.payment_failed (card declined)
6
Webhook updates Supabase
Vercel
Supabase
The webhook handler updates the claims table in Supabase to mark the studio as paid. It stores the Stripe customer ID and subscription ID so we can look them up for cancellations or billing issues later.
🗄️Updates: claims row where studio matches → sets status = 'paid', stores stripe_customer_id, stripe_subscription_id
🔄On cancellation: sets status back to 'verified' (they keep the claim, just lose paid features)
7
Webhook updates WordPress
Vercel
WordPress
The same webhook handler also calls the WordPress REST API to update the studio's studio_tier field from "claimed" to "paid". This is what controls what displays on the studio detail page — the lead form, featured badge, and priority sort all read from this field.
🔌Uses WP Application Password (already stored in Vercel env vars)
📝PATCH wp-json/wp/v2/dance_studio/{id} → sets studio_tier: "paid"
⚡Also fires ISR revalidate on that studio's page so the changes go live immediately
8
The studio detail page upgrades live
Vercel
Because the WP field changed and ISR fired, the studio's public page rebuilds within seconds. It now shows the Featured badge, the lead capture form, and the studio floats to the top of search results for their city/style. All automatic — no manual publish needed.
⭐"Featured" gold badge appears on their listing card in search results
📋Lead capture form appears on their studio detail page (name, email, message → sent to owner via Resend)
🔝Search results sort: paid studios appear before free/claimed studios
🖼️Enhanced profile fields unlock: logo upload, banner image, promo video embed
✓
Monthly auto-billing handles itself
Stripe
Stripe automatically charges the owner's card every 30 days with no action needed from you. If a card fails, Stripe retries 3 times then sends a cancellation webhook which downgrades the studio automatically. You see all of this in the Stripe dashboard.
🔁Stripe handles renewals, retries, receipts, and cancellations
📊You see MRR, churn, and subscriber list in Stripe dashboard — no code needed
❌On cancellation: webhook fires → Supabase + WP both updated → featured features removed automatically

How Resend Works (the vendor itself)
📤 Sending an email with Resend
From our API route, one function call:

resend.emails.send({
  from: 'info@ballroomdancedirectory.com'
  to: ownerEmail
  subject: '...'
  html: emailTemplate
})

That's it. Resend handles all the delivery infrastructure, spam scoring, and bounce handling.
🌐 Domain verification (one-time setup)
To send from info@ballroomdancedirectory.com, we add 3 DNS records in Namecheap (same place we manage the domain). Takes 5 minutes. Resend then verifies our domain ownership and enables sending. This is what makes emails land in inboxes instead of spam.
